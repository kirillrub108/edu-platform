# Локальный AI вместо облака: qwen3 + Qwen-VL + Silero и биллинг по кредитам

> **Площадка:** vc.ru · **Теги:** AI, SaaS, экономика продукта, приватность, импортонезависимость · **Время чтения:** ~9 мин

**TL;DR.** Edllm генерирует озвученные видеоуроки из PPTX — а это три AI-операции: LLM режет сценарий по слайдам, vision-модель «читает» слайды глазами, TTS озвучивает. Мы сделали так, что весь этот стек можно крутить **локально**, без облака: `qwen3` для текста, локальная `Qwen-VL` для слайдов, Silero для голоса. Смена провайдера — это правка `.env`, а не кода. А расход считается не «квотами на запросы», а кредитами с резервированием. Рассказываем про приватность, экономику и почему это вообще возможно.

---

## Зачем тащить AI «к себе»

У облачного AI два неудобных свойства для образовательного SaaS.

Первое — **деньги**. Каждая генерация урока — это десятки vision-вызовов (по слайду) плюс LLM плюс синтез речи. На бесплатных и пробных пользователях облачный счёт капает ещё до того, как кто-то заплатил.

Второе — **данные**. Лекции, презентации, материалы курсов — это интеллектуальная собственность преподавателя. Отправлять её в чужое облако хочется не всем, а кому-то и нельзя по требованиям.

Поэтому мы с самого начала проектировали стек так, чтобы он **мог быть полностью локальным** — и при этом не превращался в две разные кодовые базы «облако» и «не облако».

---

## Один протокол — и провайдер меняется в `.env`

Ключевое архитектурное решение: **все наши AI-провайдеры говорят на OpenAI-совместимом протоколе.** Локальный Ollama, облачный шлюз, YandexGPT — у всех один и тот же интерфейс `chat.completions`. Значит, сервису всё равно, к кому он подключён: меняются только `base_url`, `model` и ключ.

Текстовая LLM, которая режет сценарий лекции на куски по слайдам:

```python
# backend/app/services/llm_service.py
class LLMService:
    def __init__(self) -> None:
        self.client = AsyncOpenAI(
            base_url=settings.LLM_BASE_URL,   # локальный дефолт: Ollama на хосте
            api_key=settings.LLM_API_KEY,
            timeout=LLM_REQUEST_TIMEOUT_SECONDS,
            max_retries=LLM_MAX_RETRIES,
        )
        self.model = settings.LLM_MODEL       # семейство qwen3 (тег задаётся через env)
```

Хочешь облако — поменял три переменные окружения, код не тронут. Хочешь self-hosted — направил `LLM_BASE_URL` на локальный Ollama. Тот же приём — для vision и для TTS. Сервис провайдер-агностичен by design.

## Vision вместо OCR: модель «смотрит» на слайд

Самое интересное — как мы понимаем содержимое слайдов. Можно было пойти путём OCR: выдрать текст, распознать. Но слайд — это не только текст: это схемы, графики, расположение, акценты. Мы используем **vision-LLM**, которая буквально получает картинку слайда и описывает её словами:

```python
# backend/app/services/vision_analysis.py
class SlideVisionAnalyzer:
    def __init__(self) -> None:
        self.provider = (settings.VISION_PROVIDER or "ollama").lower()
        if self.provider == "ollama":
            self._ollama_client = AsyncOpenAI(
                base_url=settings.VISION_OLLAMA_BASE_URL,   # локальный Ollama на хосте
                api_key=settings.VISION_API_KEY,
            )
            self._model = settings.VISION_MODEL             # локальная Qwen-VL (7B)
```

Картинка уходит в модель как `data:image/jpeg;base64,...` — обычным `image_url`, как в OpenAI Vision API. На выходе — текстовое описание слайда, которое дальше идёт в озвучку и сценарий.

Vision-вызовы дорогие, поэтому результат **кэшируется по содержимому**. Ключ кэша — sha256 байтов картинки плюс провайдер и модель: один и тот же слайд второй раз не анализируется, а смена модели автоматически инвалидирует кэш.

```python
def _cache_key(self, image_path: str) -> str:
    """Стабильный ключ: sha256(байты файла) + провайдер/модель."""
    h = hashlib.sha256()
    with open(image_path, "rb") as f:
        for block in iter(lambda: f.read(64 * 1024), b""):
            h.update(block)
    h.update(b"|"); h.update(self.provider.encode())
    h.update(b"|"); h.update((self._model or "").encode())
    return h.hexdigest()
```

Перегенерация урока, повтор того же слайда в другом курсе — всё это **бесплатно**, потому что vision-результат уже лежит на диске.

## Голос: Silero в контейнере — синтез без счёта

Озвучку по умолчанию делает **Silero TTS**, который едет как обычный контейнер рядом с приложением. Интерфейс — единый, как и у LLM:

```python
# backend/app/services/tts_service.py
def synthesize(self, text, output_path, voice=None) -> str:
    provider = settings.TTS_PROVIDER
    if provider == "silero":
        return self._synthesize_silero(text, output_path, ...)   # self-hosted
    elif provider == "polza":
        return self._synthesize_polza(text, output_path, ...)    # облако
    ...
```

И тут видно, как локальность влияет на экономику напрямую: self-hosted Silero **не порождает счёт провайдера**, и мы это явно отмечаем в журнале использования:

```python
# self-hosted синтез — без биллингового счёта провайдера
usage_service.record_tts_usage("silero", len(plain), billable=False)
```

То есть у нас есть журнал стоимости (`generation_usage`) для расчёта маржи — но локальный TTS в нём проходит как «ноль провайдерских рублей». Считаем мы при этом только то, за что реально платим внешнему сервису.

## Биллинг: кредиты с резервированием, а не квоты на запросы

Теперь — как пользователь за это платит. Мы сознательно отказались от «квот на число запросов» в пользу **кредитов**. Причина — длинные асинхронные операции, которые могут упасть на середине. Если просто «списать за запуск», то на упавшей задаче пользователь теряет деньги, а на двойном ретрае мы можем списать дважды.

Поэтому модель — **резервируй → финализируй**:

```python
# backend/app/services/billing_service.py
async def reserve_credits(db, user_id, amount, ref_id, operation) -> bool:
    account = await db.scalar(
        select(CreditAccount).where(CreditAccount.owner_id == user_id).with_for_update()
    )
    if account.balance - account.reserved < amount:
        await db.rollback()
        return False                       # не хватает — не бросаем, решает вызывающий
    account.reserved += amount             # держим хелд под операцию
    db.add(CreditTransaction(operation=CreditOperation.RESERVE, ref_id=ref_id, ...))
    await db.commit()
    return True

async def charge_credits(db, user_id, amount, ...):
    """Финализация: списать с баланса и освободить хелд."""
    account.balance  = max(0, account.balance  - amount)
    account.reserved = max(0, account.reserved - amount)
# release_credits() — возвращает хелд, не трогая баланс (если операция не состоялась)
```

Перед запуском видео/vision/квиза кредиты **резервируются** под строковой блокировкой (`SELECT ... FOR UPDATE`), а по факту завершения — **списываются** (`charge`) или **возвращаются** (`release`). Баланс не уйдёт в минус и не спишется дважды.

## Сколько что стоит — и почему так

Цены операций не разбросаны по коду, а лежат в одном месте. Плоские операции — в весах, видео — по формуле:

```python
# backend/app/constants.py
CREDIT_WEIGHTS = {
    "vision_analyze": 5, "slide_regen": 1,
    "quiz_generate": 5,  "ai_review": 2,
    "quiz_grade": 0,                    # грейдинг бесплатный — маркетинг
}
VIDEO_TEXT_BASE_CREDITS = 2
VIDEO_AUTO_BASE_CREDITS = 3
TTS_CHARS_PER_CREDIT    = 3000
# текстовый режим: база + слайды + ceil(символы_сценария / TTS_CHARS_PER_CREDIT)
```

Логика прозрачна: чем длиннее лекция (больше символов на озвучку), тем больше кредитов. Грейдинг студенческих ответов стоит ноль — это осознанно бесплатная фича.

Бесплатные аккаунты получают не «приветственные кредиты», а **пожизненный триал** — фиксированное число уроков и квизов на весь срок жизни аккаунта (учитывается в счётчиках с периодом `lifetime`). Планы дают месячный лимит с переносом части остатка на следующий месяц:

```python
TRIAL_LECTURES = 2; TRIAL_QUIZZES = 2        # на весь срок жизни аккаунта
CREDIT_CARRYOVER_RATIO = 0.5                  # до 50% месячного лимита переносится
PLAN_CONFIGS = {
    "free":    {"monthly_allowance": 0,   ...},
    "starter": {"monthly_allowance": 30,  ...},
    "pro":     {"monthly_allowance": 120, ...},
    "school":  {"monthly_allowance": 500, ...},
}
```

*(Конкретные числа — иллюстративные, действующая тарифная сетка живёт в продукте.)*

Под капотом крутится и журнал провайдерской стоимости в рублях — он питает мониторинг маржи. Именно он показывает разницу между «крутим на Silero локально» (провайдерский счёт ≈ 0) и «ушли в облако» — и эта разница и есть экономический смысл локального стека.

## Грабли локального стека (честно)

Локальность — это не бесплатный обед. Из нашего списка известных проблем:

- **`host.docker.internal` не работает на Linux по умолчанию.** Локальный дефолт указывает на Ollama по этому адресу — и на Linux-хосте без отдельного маппинга он молча не резолвится.
- **Ollama должна быть на хосте — автоматизации нет.** Ничто не скачивает и не поднимает qwen-модели за вас; нет провайдера — генерация падает на шаге LLM-нарезки или vision-разбора.
- **LLM иногда возвращает не N чанков.** Если модель не отдала ровно по куску на слайд, включается фолбэк-нарезка по предложениям — и качество озвучки проседает.
- **Конкуренция vision захардкожена.** Разбор слайдов идёт с фиксированным семафором, а не подстраивается под пропускную способность провайдера.
- **Кэши растут без сборки мусора.** `slides_cache/` и `summaries_cache/` ключуются по хешу содержимого и безопасны к удалению, но автоматически не чистятся.

Это не «недоделки», а честный трейд-офф: за приватность и нулевую маржинальную стоимость платишь тем, что инфраструктуру AI держишь и обслуживаешь сам.

---

## Что в итоге

- **Провайдер меняется правкой `.env`** — облако и self-hosted это один и тот же код.
- **Vision вместо OCR** — модель понимает слайд целиком, а не только его текст.
- **Кэш по содержимому** делает повторные vision-вызовы бесплатными.
- **Локальный Silero** убирает счёт за озвучку — а журнал маржи это честно показывает.
- **Кредиты с резервированием** не дают балансу уйти в минус или списаться дважды на упавшей задаче.

Главный смысл локального стека — не «мы против облака», а **свобода выбора без переписывания кода**. Один протокол на всех провайдеров превращает стратегическое решение «облако или своё железо, приватность или удобство» в одну строку конфига. А это ровно та гибкость, которую SaaS обычно теряет, слишком рано прибив гвоздями одного вендора.
